# UI プレビューの見方（GitHub Pages）

このリポジトリに**ブラウザで動くデモアプリ**を同梱してあります。下記の手順で、2分ほどで確認できるようになります。

> **やること**: GitHub の設定画面で1箇所だけスイッチを入れる。それだけです。

---

## 1. GitHub Pages を有効化する

1. ブラウザで以下の URL を開く（要ログイン）:
   
   👉 **https://github.com/sekiemon1990/claude-code/settings/pages**

2. **Build and deployment** の項目で:

   - **Source**: `Deploy from a branch` を選択
   - **Branch**: `claude/sales-recording-app-9LcDo` / `/docs` を選択
   - **Save** をクリック

   画面イメージ:
   ```
   ┌──────────────────────────────────────────────────┐
   │ Build and deployment                             │
   │                                                  │
   │ Source:  [ Deploy from a branch ▼ ]             │
   │                                                  │
   │ Branch:  [ claude/sales-recording-app-9LcDo ▼ ] │
   │          [ /docs ▼ ]     [ Save ]                │
   └──────────────────────────────────────────────────┘
   ```

3. Save を押すと、画面上部に「Your site is live at ...」と出るまで **1〜2分**ほど待つ

---

## 2. デモページを開く

以下の URL にアクセス:

👉 **https://sekiemon1990.github.io/claude-code/**

（`/docs/index.html` から自動で `/docs/demo/` へ遷移します）

---

## 3. デモで確認できる操作

- ✅ 自動ログイン（「デモ営業マン」として即ログイン済み）
- ✅ **一覧画面**（右上に赤い `DEMO` バッジ）
- ✅ **「+ 新規録音」ボタン**
- ✅ **案件選択画面**（モックの3件：田中様 / 山田様 / 佐藤様）
- ✅ **録音画面**
  - タイマーが進む（タイマーのみ。実マイクは動きません）
  - 停止して保存
- ✅ **一覧で処理ステータスが自動で進む**
  - `送信中 15%→40%→70%→100%`（プログレスバー）
  - `文字起こし中`（黄色バッジ）
  - `議事録生成中`
  - `完了`（緑バッジ）
  - **完了まで合計 約8秒**で自動遷移します
- ✅ **行をタップ → 詳細画面**
  - 案件情報カード
  - 疑似的な文字起こし（営業マンとお客様の対話8ターン分）
  - 疑似議事録（サマリ / お客様情報 / 査定品目 / 提示額28万円 / 次回アクション）

## 4. デモで再現**しない**もの

- 実マイクでの録音・音声再生（タイマーだけ動く）
- マクサスコアからのディープリンク起動（実機ビルド時のみ）
- バックグラウンド送信（アプリを閉じた時）
- 端末ストレージ残量の実測

これらは実機ビルド（`docs/SETUP.md` 参照）で確認できます。

## トラブルシューティング

### 「404 Not Found」が出る
GitHub Pages のビルドが完了していない可能性があります。
- https://github.com/sekiemon1990/claude-code/actions で `pages-build-deployment` の Actions を確認
- 緑のチェックマークが付くまで待つ（通常1〜2分）

### 画面が真っ白
- ブラウザの開発者ツール（F12）でエラーがないか確認
- キャッシュが古い可能性があるので Ctrl+Shift+R（Mac: Cmd+Shift+R）で強制リロード

### ログイン画面から進まない
- このデモは自動ログイン設定なので本来はログイン画面が出ない
- もしログイン画面が出た場合はビルド設定のミス。ご連絡ください
